Git Command Cheatsheet

A quick reference guide for the most commonly used Git commands. Bookmark this page for easy access.

Setup & Configuration
git config --global user.name "name"

Set the name attached to your commits

git config --global user.name "John Doe"

git config --global user.email "email"

Set the email attached to your commits

git config --global user.email "john@example.com"

git config --list

View all Git configuration settings

git init

Create a new local Git repository

git clone <url>

Clone a repository from a remote source

git clone https://github.com/user/repo.git

Basic Commands
git status

Show the status of your working directory

git add <file>

Stage a specific file for commit

git add index.html

git add .

Stage all changed files for commit

git commit -m "message"

Commit staged changes with a message

git commit -m "Add login feature"

git commit --amend

Modify the most recent commit

Branching
git branch

List all local branches

git branch <name>

Create a new branch

git branch feature-login

git checkout <branch>

Switch to a different branch

git checkout main

git checkout -b <name>

Create and switch to a new branch

git checkout -b feature-signup

git switch <branch>

Switch branches (newer command)

git switch -c <name>

Create and switch to new branch (newer)

git branch -d <name>

Delete a branch (safe)

git branch -D <name>

Force delete a branch

Merging & Rebasing
git merge <branch>

Merge a branch into current branch

git merge feature-login

git rebase <branch>

Rebase current branch onto another

git rebase main

git rebase --abort

Cancel an in-progress rebase

git merge --abort

Cancel an in-progress merge

git cherry-pick <commit>

Apply a specific commit to current branch

History & Inspection
git log

Show commit history

git log --oneline

Show compact commit history

git log --graph

Show history with branch graph

git log -p

Show history with file changes

git show <commit>

Show details of a specific commit

git diff

Show unstaged changes

git diff --staged

Show staged changes

git blame <file>

Show who changed each line of a file

Remote Repositories
git remote -v

List all remote repositories

git remote add <name> <url>

Add a new remote repository

git remote add origin https://github.com/user/repo.git

git push <remote> <branch>

Push commits to remote repository

git push origin main

git push -u origin <branch>

Push and set upstream branch

git pull

Fetch and merge changes from remote

git fetch

Download changes without merging

git push --force

Force push (use with caution!)

Undoing Changes
git restore <file>

Discard changes in working directory

git restore --staged <file>

Unstage a file (keep changes)

git reset HEAD~1

Undo last commit (keep changes)

git reset --hard HEAD~1

Undo last commit (discard changes)

git revert <commit>

Create new commit that undoes changes

git clean -fd

Remove untracked files and directories

Stashing
git stash

Temporarily save uncommitted changes

git stash save "message"

Stash with a description

git stash list

List all stashed changes

git stash pop

Apply and remove latest stash

git stash apply

Apply latest stash (keep in list)

git stash drop

Delete the latest stash

git stash clear

Delete all stashes

Tags
git tag

List all tags

git tag <name>

Create a lightweight tag

git tag v1.0.0

git tag -a <name> -m "msg"

Create an annotated tag

git tag -a v1.0.0 -m "Release 1.0"

git push origin <tag>

Push a specific tag to remote

git push origin --tags

Push all tags to remote

.gitignore Patterns
*.log

Ignore all .log files

node_modules/

Ignore the node_modules directory

/build

Ignore build folder in root only

*.env

Ignore environment files

!important.log

Don't ignore this specific file

**/*.tmp

Ignore .tmp files in all directories

Helpful Aliases
git config --global alias.co checkout

Use "git co" instead of "git checkout"

git config --global alias.br branch

Use "git br" instead of "git branch"

git config --global alias.ci commit

Use "git ci" instead of "git commit"

git config --global alias.st status

Use "git st" instead of "git status"

git config --global alias.lg "log --oneline --graph"

Create a pretty log alias
